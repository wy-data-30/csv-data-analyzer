# Smart CSV Analyzer

Smart CSV Analyzer 是一个纯前端通用 CSV 数据分析工具。用户上传任意结构化 CSV 文件后，系统会在浏览器端完成数据预览、字段类型识别、数据质量检查、描述性统计、可视化图表和基础分析结论生成。

项目不依赖后端、数据库或登录系统，适合用于轻量级数据探索、教学演示、数据质量检查和前端数据分析能力展示。

## 项目解决的问题

日常工作中，CSV 文件来源广、结构不固定，用户通常需要先手动判断字段类型、检查缺失值、统计指标、再制作基础图表。Smart CSV Analyzer 将这些重复步骤自动化：

- 快速理解陌生 CSV 的字段结构和数据规模。
- 自动发现缺失值、重复行和数值异常值。
- 为数值、分类、日期字段生成基础统计和图表。
- 支持用户手动映射字段，避免依赖固定字段名。
- 针对销售、成绩、二手商品、问卷、用户行为等常见场景生成谨慎的数据事实结论。

## 核心功能

- CSV 文件上传与浏览器端解析。
- 前 10 行数据预览。
- 自动识别数值字段、分类字段、日期字段和 ID 字段。
- 数据概览：总行数、总字段数、字段类型数量、缺失值总数、重复行数量。
- 数据质量报告：每列缺失值数量、缺失率、重复行检查、数值异常值检查。
- 描述性统计：平均值、中位数、最大值、最小值、标准差。
- 分类字段频数统计：Top 10 类别和占比。
- 自动图表：数值分布图、分类 Top 10 柱状图、日期趋势图。
- 自定义分析：选择数值指标、分类维度和可选时间字段，生成分组汇总和趋势图。
- 场景模板：通用数据分析、销售数据分析、成绩数据分析、二手商品价格分析、问卷调查分析、用户行为分析。
- 基础分析结论：仅基于 CSV 数据事实生成，不编造业务背景。
- 响应式布局：桌面端双列图表，移动端单列展示，表格支持横向滚动。

## 技术栈

- HTML5
- CSS3
- Vanilla JavaScript
- PapaParse：通过 CDN 解析 CSV
- Chart.js：通过 CDN 生成图表

所有分析逻辑均在浏览器端执行。

## 如何本地运行

方式一：直接打开页面

```text
index.html
```

方式二：启动本地静态服务

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

建议使用方式二。部分浏览器会限制本地 `fetch`，可能导致“加载示例数据”按钮无法直接读取 `sample-data.csv`。

## 如何使用测试数据

项目内置 6 个测试 CSV 文件，均可直接上传到页面测试：

- `sample-data.csv`：通用订单数据，适合测试完整分析流程。
- `sample-sales.csv`：销售数据，适合测试销售额、单价、数量、品类、地区和日期趋势。
- `sample-students.csv`：学生成绩数据，适合测试成绩分布、班级/科目分组和考试日期趋势。
- `sample-used-products.csv`：二手商品价格数据，适合测试价格分布、平台、城市、成色和异常价格。
- `sample-survey.csv`：问卷调查数据，适合测试满意度、年龄、性别、城市和选择题频数。
- `sample-user-behavior.csv`：用户行为数据，适合测试用户 ID、事件、渠道、设备和行为时间趋势。

这些测试文件包含中文字段名和中文字段值，采用 UTF-8 BOM 编码，并刻意加入少量缺失值、重复行和异常值，用于验证数据质量检查能力。

## 支持的数据类型

- 数值字段：金额、数量、分数、价格、时长、比例等可解析为数字的数据。
- 分类字段：地区、品类、城市、渠道、设备、性别、科目等离散文本数据。
- 日期字段：可被浏览器解析为日期的字段，如 `2025-01-01` 或 `2025-06-01 08:00:00`。
- ID 字段：订单号、用户 ID、学生 ID、事件 ID、编码、主键等标识类字段。

## 字段识别规则

系统会对每一列计算非空值数量、唯一值数量、唯一值比例、数值匹配率和日期匹配率，并按以下规则识别字段类型：

- ID 字段：字段名包含 `id`、`uuid`、`code`、`key`、`number`、`no`、`编号`、`编码`、`订单号`、`账号`、`工号`、`主键` 等关键词；或唯一值比例很高且不主要表现为数值/日期。
- 日期字段：非空值中至少 80% 可被解析为日期。
- 数值字段：非空值中至少 80% 可被解析为数字。
- 分类字段：不满足上述规则时，默认识别为分类字段。

字段识别是启发式规则，用户仍可在自定义分析和场景模板中手动选择字段。

## 数据质量检查规则

- 缺失值：空字符串、`null`、`undefined` 或仅包含空白字符的值会被视为缺失。
- 缺失率：每列缺失值数量除以总行数。
- 重复行：按所有字段的标准化值组合进行整行重复判断。
- 数值异常值：对数值字段使用 IQR 方法检测异常值，即低于 `Q1 - 1.5 * IQR` 或高于 `Q3 + 1.5 * IQR` 的值。
- 空 CSV：没有表头或没有有效数据行时，会显示清晰错误提示，不展示旧分析结果。

## 图表说明

- 数值字段分布图：使用直方图展示数值字段的区间分布。
- 分类字段 Top 10 柱状图：展示分类字段出现频数最高的 10 个类别。
- 日期字段趋势图：按日期聚合记录数或数值字段汇总值，生成趋势折线图。
- 自定义分组图：用户选择一个数值字段和一个分类字段后，生成按分类维度聚合的柱状图。
- V2 字段选择分析：生成按分类字段分组的求和结果、平均值结果和 Top 10 分组柱状图；选择日期字段后生成时间趋势图。
- 场景模板图表：根据不同模板和字段映射生成对应的主图、辅助图和可选趋势图。

## 项目亮点

- 通用 CSV 分析，不绑定固定业务字段名。
- 纯前端实现，数据不上传服务器。
- 支持中文字段名和中文字段值。
- 自动分析与手动字段映射结合，兼顾易用性和灵活性。
- 覆盖数据预览、字段识别、质量检查、统计分析、可视化和分析结论。
- 内置多场景测试数据，便于演示和回归测试。
- 页面风格简洁专业，适配桌面和移动端。

## 项目局限

- 依赖浏览器内存，不适合直接分析超大 CSV 文件。
- 日期解析依赖浏览器 `Date` 能力，非标准日期格式可能无法准确识别。
- 字段类型识别基于启发式规则，复杂业务字段仍需要用户手动确认。
- 当前只提供基础统计分析，不包含机器学习、预测建模或高级数据清洗。
- PapaParse 和 Chart.js 通过 CDN 加载，离线环境需要改为本地依赖。
- 分析结果不会持久化，刷新页面后需要重新上传 CSV。
