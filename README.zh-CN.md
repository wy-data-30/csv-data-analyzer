# Smart Tabular Analyzer

**当前版本：** V2.3

**中文名称：** 智能表格数据分析工具

**副标题：** Browser-based CSV and Excel data analysis tool

**GitHub 仓库：** [github.com/wy-data-30/smart-tabular-analyzer](https://github.com/wy-data-30/smart-tabular-analyzer)

**在线演示：** [打开 Smart Tabular Analyzer](https://wy-data-30.github.io/smart-tabular-analyzer/)

[English README](./README.md)

Smart Tabular Analyzer（智能表格数据分析工具）是一个纯前端 CSV/Excel 数据分析工具，用于快速查看和分析结构化数据文件。用户上传文件后，应用会在浏览器端完成数据预览、字段类型识别、数据质量检查、描述性统计、图表生成、自定义分组分析和基础结论生成。

该项目是静态网站，不依赖后端服务、数据库、账号系统或构建流程，适合部署到 GitHub Pages 等静态托管平台。

## 项目简介

CSV 文件经常来自不同系统或业务场景，字段结构并不固定。Smart Tabular Analyzer 通过自动识别字段类型和生成基础分析结果，帮助用户快速了解数据规模、字段结构、缺失情况、重复行、异常值、分类分布和时间趋势。

项目不会假设固定字段名。对于不同结构的数据，用户可以通过字段选择器或场景模板手动映射分析字段。

## 功能特性

- 在浏览器端上传并解析 CSV 文件。
- 支持通过 SheetJS 导入 Excel `.xlsx` / `.xls` 文件；单工作表文件直接分析，多工作表文件先显示工作表选择器。
- 支持手动选择 CSV 文件编码，包括自动识别、UTF-8、GBK 和 GB18030。
- 展示数据集前 10 行预览。
- 自动识别数值字段、分类字段、日期字段和 ID 字段，并针对学号/记录编号及年份字段进行语义识别。
- 文件解析后展示字段配置表，包括字段名、自动识别类型、非空值数量、唯一值数量和示例值。
- 支持将字段暂存修正为数值、分类、日期、ID 或忽略，点击“应用字段配置”后统一重算统计、趋势、图表和结论。
- 数值或日期字段存在无法安全转换的非空值时显示失败数量，并继续使用有效值分析。
- 支持一键恢复全部自动识别结果。
- 生成数据概览，包括行数、字段数、字段类型数量、缺失值总数和重复行数量。
- 支持同时按分类多选、数值范围和可选日期范围筛选当前分析数据，不修改导入的原始记录。
- 生成数据质量报告，包括每列缺失值、缺失率、重复行和数值异常值。
- 为数值字段计算平均值、中位数、最大值、最小值和标准差；整列使用 `%` 的字段在内部仍按小数计算，但在统计表和图表中按百分比显示。
- 为分类字段生成 Top 10 频数统计和占比。
- 自动生成数值分布图、分类 Top 10 柱状图和日期趋势图。
- 支持用户选择数值指标、分类分组字段和可选日期字段进行自定义分析。
- 内置通用数据、销售数据、学生成绩、二手商品价格、问卷调查和用户行为等场景模板。
- 基于数据事实生成谨慎的基础分析结论，不编造业务背景。
- 支持将当前筛选结果或原始数据去除完全重复行后的结果导出为 CSV；保留原始字段顺序，并使用 UTF-8 BOM 兼容 Excel 中文显示。
- 分析完成后可导出 HTML 或 Markdown 报告，两种格式均包含来源信息、字段类型、数据质量、统计趋势、自定义分组结果和自动结论。
- HTML 报告会嵌入当前图表，可作为独立文件离线打开。
- 支持桌面端和移动端响应式布局，表格区域可横向滚动。

## 本地运行

可以直接用浏览器打开 `index.html`。

更推荐在项目目录下启动一个本地静态服务：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

使用本地静态服务可以避免部分浏览器限制本地文件 `fetch()`，从而影响“加载示例数据”按钮读取 `sample-data.csv`。

## 使用方法

1. 在浏览器中打开应用。
2. 上传 CSV 时，可根据需要选择自动识别、UTF-8、GBK 或 GB18030；该编码选项仅用于 CSV。
3. 通过文件选择或拖拽上传 `.csv`、`.xlsx` 或 `.xls` 文件。
4. 如果 Excel 工作簿包含多个工作表，先选择目标工作表，再点击“分析所选工作表”；单工作表文件会直接进入分析。
5. 查看字段配置表，将字段修正为数值、分类、日期、ID 或忽略，然后点击“应用字段配置”。
6. 查看转换失败数量；无法安全转换的数值或日期内容会被跳过，不会中断整个文件分析。
7. 如需撤销人工修正，点击“恢复自动识别”，系统会按最初推断结果重新计算。
8. 查看数据概览、质量报告、统计结果、图表和分析结论。
9. 根据需要同时应用分类、数值和日期筛选条件，确定当前分析范围。
10. 在“导出数据”区域下载当前筛选结果，或下载原始数据去除完全重复行后的结果。
11. 在自定义分析区域选择字段，或使用场景模板生成分析。
12. 分析完成后，在报告目录中点击“导出 HTML 报告”或“导出 Markdown 报告”。有效分析生成前，报告导出按钮保持不可用。

应用不要求固定字段名。场景模板分析完全基于用户选择的字段生成。

## 字段类型确认与修正

- 下拉框修改会先暂存，不会立即改变当前分析；点击“应用字段配置”后才统一生效。
- 应用配置会复用同一分析链路，重新生成描述性统计、分类频数、日期趋势、图表、字段选择器、场景模板映射和自动结论。
- 数值和日期转换沿用分析模块的严格转换规则；空值不计入转换失败，非空但无效的值会计数并从对应分析中跳过。
- 设为“忽略”的字段仍保留在原始数据预览和数据质量报告中，但不参与统计、趋势、图表、场景模板和自动结论。
- “恢复自动识别”会清除所有人工覆盖，不需要重新解析源文件。

CSV 中的中文字段名或内容出现乱码时，可以在上传前选择 `GBK` 或 `GB18030`，也可以在 Excel/WPS 中另存为 `CSV UTF-8`。Excel 工作簿由 SheetJS 解析，中文工作表名、字段名和单元格内容会直接保留。

## 字段识别补充规则

- ID：除 `id`、`code`、`编号`、`编码`、`订单号` 等关键词外，也识别 `学号`、`学籍号`、`考生号`、`准考证号`、`设备号` 等标识字段。纯数字学号不会进入数值统计；`学号数量` 等含义不同的字段仍保持数值类型。
- 年份：`年份`、`年度`、`学年`、`财年`、`入学年`、`毕业年` 等语义明确的列，可将 1900—2200 范围内的四位年份识别为日期并按 `YYYY` 展示趋势。普通四位数列不会仅因数值像年份就自动改为日期。
- 日期：支持严格校验的年优先格式、中文年月日以及日优先/月优先格式。系统会利用同列中不含歧义的日期判断日/月顺序；如果同一列同时出现互相冲突的顺序，只转换含义明确的单元格，并把歧义单元格计入转换失败数量。
- 价格与百分比：价格中的 `￥`、`¥`、`$`、千位逗号等会在计算时安全清理。只有当整列所有可转换的非空数值都带 `%` 时才按百分比展示；百分比与普通小数混用的列保持普通数值格式，避免错误放大。

## 处理后数据导出

“导出数据”区域提供两种完全在浏览器本地生成的 CSV 文件：

- **筛选后 CSV：** 导出当前 `filteredRows` 中的记录；未应用筛选时包含全部导入记录。
- **去重后 CSV：** 基于原始导入数据的全部原始字段识别完全重复行，每组重复记录保留首次出现的一行；该结果不受当前筛选条件影响。

两种导出都会保留原始字段顺序和单元格内容，使用逗号分隔、CRLF 换行和 UTF-8 BOM，确保中文字段名和中文内容能够在 Excel/WPS 中正常打开。文件名格式为 `原文件名_filtered_YYYYMMDD.csv` 或 `原文件名_deduplicated_YYYYMMDD.csv`，例如 `sales_filtered_20260718.csv`。

导出操作只会创建新的浏览器下载文件，不会修改或覆盖用户上传的 CSV/Excel 文件，也不会向服务器发送任何原始记录。

## 分析报告导出

V2.3 支持将当前分析下载为独立 HTML 文件或 Markdown 文件。报告的整理、生成和下载全部在当前浏览器中完成，原始数据行与报告内容不会上传。

两种格式均包括：

- 原始文件名称；如果导入的是 Excel，还包括当前选择的工作表名称。
- 分析时间以及数据行数、字段数。
- 实际应用的字段类型结果，并在适用时记录忽略字段和转换失败数量。
- 数据质量汇总、各字段缺失情况、重复行和数值异常值结果。
- 可用数值字段的描述性统计。
- 可用分类字段的 Top 10。
- 可用日期字段的趋势摘要。
- 存在有效指标和分组选择时的当前自定义分组分析结果。
- 自动生成的分析结论。

HTML 报告会把当前页面已生成的图表和报告展示样式嵌入下载文件，因此报告文件本身无需联网即可打开。Markdown 报告采用标题、列表和表格保存文字分析结果，方便在编辑器、知识库或版本管理系统中继续使用。

下载文件名由原文件基础名称和本地导出日期时间组成，例如 `sales-20260717-153045.html` 或 `sales-20260717-153045.md`。对于空文件或无效数据，分析不会完成，两个导出按钮保持不可用。对于有效但没有数值字段、分类字段、日期字段或自定义分组结果的数据，报告会明确记录“无适用字段/结果”，不会因此导出失败。

## 数据隐私说明

所有 CSV 和 Excel 解析、筛选、处理后数据导出、分析及报告导出都在浏览器本地完成。应用不会将用户上传的数据文件或生成的报告发送到服务器，也不会存储用户文件或将数据内容发送给第三方 API。

页面会从 CDN 加载 PapaParse、Chart.js 和 SheetJS。浏览器在加载这些外部脚本时会访问 CDN 域名，但上传的数据文件内容仍保留在浏览器运行环境内。

## 已知限制

- 大型 CSV 或 Excel 文件受浏览器内存和单页面运行性能限制。
- 日期识别采用严格格式校验；含义不明确的月/日值需要同列中存在一致的无歧义证据。若日优先与月优先证据冲突，歧义值会保留为转换失败并提示用户确认。
- Excel 分析要求所选工作表的第一个非空行为唯一、非空的文本表头；空表、无表头、受保护或结构异常的工作表可能无法导入，并会显示错误提示。
- 字段类型识别基于启发式规则，含义模糊的字段仍需要用户手动确认。
- 人工指定数值或日期类型不会改写源数据；严格转换失败的值会从对应分析中排除，并显示失败数量。
- 当前项目聚焦基础探索式分析，不包含高级数据清洗、预测建模或机器学习能力。
- 分析结果不会持久化，刷新页面后需要重新上传数据文件。
- CDN 依赖需要网络访问；离线环境需要将依赖下载后改为本地引用。
