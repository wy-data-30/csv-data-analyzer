# Smart CSV Analyzer

**当前版本：** V2.1

**在线演示：** [打开 Smart CSV Analyzer](https://wy-data-30.github.io/csv-data-analyzer/)

[English README](./README.md)

Smart CSV Analyzer 是一个纯前端 CSV/Excel 数据分析工具，用于快速查看和分析结构化数据文件。用户上传文件后，应用会在浏览器端完成数据预览、字段类型识别、数据质量检查、描述性统计、图表生成、自定义分组分析和基础结论生成。

该项目是静态网站，不依赖后端服务、数据库、账号系统或构建流程，适合部署到 GitHub Pages 等静态托管平台。

## 项目简介

CSV 文件经常来自不同系统或业务场景，字段结构并不固定。Smart CSV Analyzer 通过自动识别字段类型和生成基础分析结果，帮助用户快速了解数据规模、字段结构、缺失情况、重复行、异常值、分类分布和时间趋势。

项目不会假设固定字段名。对于不同结构的数据，用户可以通过字段选择器或场景模板手动映射分析字段。

## 功能特性

- 在浏览器端上传并解析 CSV 文件。
- 支持通过 SheetJS 导入 Excel `.xlsx` / `.xls` 文件；单工作表文件直接分析，多工作表文件先显示工作表选择器。
- 支持手动选择 CSV 文件编码，包括自动识别、UTF-8、GBK 和 GB18030。
- 展示数据集前 10 行预览。
- 自动识别数值字段、分类字段、日期字段和 ID 字段。
- 支持人工调整字段类型，并自动重算统计、图表和字段选择器。
- 生成数据概览，包括行数、字段数、字段类型数量、缺失值总数和重复行数量。
- 生成数据质量报告，包括每列缺失值、缺失率、重复行和数值异常值。
- 为数值字段计算平均值、中位数、最大值、最小值和标准差。
- 为分类字段生成 Top 10 频数统计和占比。
- 自动生成数值分布图、分类 Top 10 柱状图和日期趋势图。
- 支持用户选择数值指标、分类分组字段和可选日期字段进行自定义分析。
- 内置通用数据、销售数据、学生成绩、二手商品价格、问卷调查和用户行为等场景模板。
- 基于数据事实生成谨慎的基础分析结论，不编造业务背景。
- 支持导出独立 HTML 报告，以及通过浏览器打印功能保存 PDF。
- 支持桌面端和移动端响应式布局，表格区域可横向滚动。

## 技术栈

## 本地运行

可以直接用浏览器打开 `index.html`。

更推荐在项目目录下启动一个本地静态服务：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

使用本地静态服务可以避免部分浏览器限制本地文件 `fetch()`，从而影响“加载示例数据”按钮读取 `sample-data.csv`。

## 使用方法

1. 在浏览器中打开应用。
2. 上传 CSV 时，可根据需要选择自动识别、UTF-8、GBK 或 GB18030；该编码选项仅用于 CSV。
3. 通过文件选择或拖拽上传 `.csv`、`.xlsx` 或 `.xls` 文件。
4. 如果 Excel 工作簿包含多个工作表，先选择目标工作表，再点击“分析所选工作表”；单工作表文件会直接进入分析。
5. 查看系统生成的数据概览、字段识别、质量报告、统计结果、图表和分析结论。
6. 在自定义分析区域选择数值指标、分组字段和可选日期字段。
7. 如果数据符合某个场景，可以选择场景模板并手动映射字段后生成模板分析。

应用不要求固定字段名。场景模板分析完全基于用户选择的字段生成。

CSV 中的中文字段名或内容出现乱码时，可以在上传前选择 `GBK` 或 `GB18030`，也可以在 Excel/WPS 中另存为 `CSV UTF-8`。Excel 工作簿由 SheetJS 解析，中文工作表名、字段名和单元格内容会直接保留。

## 示例数据说明

项目内置多个示例 CSV 文件，用于测试不同分析场景：

- `sample-data.csv`：通用订单类数据，用于测试完整分析流程。
- `sample-sales.csv`：销售数据，用于测试销售额、单价、数量、品类、地区和日期趋势分析。
- `sample-students.csv`：学生成绩数据，用于测试成绩分布、班级/科目分组和考试日期趋势。
- `sample-used-products.csv`：二手商品价格数据，用于测试价格分布、平台、城市、成色和价格异常值。
- `sample-survey.csv`：问卷调查数据，用于测试满意度、年龄、性别、城市和选择题频数。
- `sample-user-behavior.csv`：用户行为数据，用于测试用户 ID、事件、渠道、设备和事件时间趋势。

示例数据采用 UTF-8 BOM 编码，包含中文字段名和中文字段值，并加入少量缺失值、重复行和异常值，用于验证数据质量检查功能。示例数据为虚构/匿名化数据。

`tests/fixtures/excel-import-test.xlsx` 是 V2.1 多工作表回归测试文件，包含中文工作表名与中文内容，并覆盖空工作表和无表头工作表场景。

## 数据隐私说明

所有 CSV 和 Excel 解析及分析都在浏览器本地完成。应用不会将用户上传的数据文件发送到服务器，也不会存储用户文件或将数据内容发送给第三方 API。

页面会从 CDN 加载 PapaParse、Chart.js 和 SheetJS。浏览器在加载这些外部脚本时会访问 CDN 域名，但上传的数据文件内容仍保留在浏览器运行环境内。

## 已知限制

- 大型 CSV 或 Excel 文件受浏览器内存和单页面运行性能限制。
- 日期识别采用严格格式校验；含义不明确的月/日格式需要人工确认。
- Excel 分析要求所选工作表的第一个非空行为唯一、非空的文本表头；空表、无表头、受保护或结构异常的工作表可能无法导入，并会显示错误提示。
- 字段类型识别基于启发式规则，含义模糊的字段仍需要用户手动确认。
- 当前项目聚焦基础探索式分析，不包含高级数据清洗、预测建模或机器学习能力。
- 分析结果不会持久化，刷新页面后需要重新上传数据文件。
- CDN 依赖需要网络访问；离线环境需要将依赖下载后改为本地引用。
