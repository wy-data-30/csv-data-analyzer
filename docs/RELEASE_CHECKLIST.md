# Smart Tabular Analyzer 发布检查清单

用于 V2.1 及后续小版本发布。按实际改动范围执行，但标记为“核心”的项目不应跳过。

## 1. 版本与范围

- [ ] 发布内容与项目定位一致：无需注册、浏览器本地处理、支持中文 CSV/Excel。
- [ ] 没有加入后端、数据库、账号、云存储或其他范围外依赖。
- [ ] `package.json`、README 当前版本和 `script.js` 报告版本保持一致。
- [ ] README、中文 README 和 `docs/` 与实际行为一致。
- [ ] 新增或修改的用户文件、测试夹具和截图不含真实个人数据。

## 2. 静态结构与依赖（核心）

- [ ] 根目录存在 `index.html`、`style.css` 和 `script.js`。
- [ ] HTML 中本地资源使用相对路径，且文件实际存在。
- [ ] PapaParse、Chart.js、SheetJS 使用固定版本 CDN。
- [ ] 三个 CDN 的 SRI 哈希与实际文件一致，`crossorigin` 配置正确。
- [ ] 页面不依赖构建步骤、后端接口或本地绝对路径。
- [ ] `node --check script.js` 通过。

## 3. 自动化测试（核心）

- [ ] `npm test` 通过。
- [ ] `npm run test:regression` 全部通过。
- [ ] 100,000 行压力测试完成，未出现明显性能退化。
- [ ] `git diff --check` 通过。
- [ ] `.github/workflows/ci.yml` 语法校验通过。
- [ ] GitHub Actions 的 CI 在目标 commit 上通过。
- [ ] 若项目增加了 `lint` script，lint 通过。

## 4. 导入回归（核心）

- [ ] UTF-8 和 UTF-8 BOM CSV 可导入，中文字段名和值正常。
- [ ] GBK 和 GB18030 CSV 可通过编码选择导入。
- [ ] `.xlsx` 和 `.xls` 可导入。
- [ ] 多工作表 Excel 可选择工作表；空表和无表头表能给出错误。
- [ ] 空 CSV、只有表头、缺失表头、重复表头和多余单元格不会静默丢失数据。
- [ ] 单列、单行、中文、Emoji、逗号、引号和换行文本可处理。
- [ ] 超过 25 MB、100,000 行或 200 列的数据会被明确拒绝。
- [ ] 连续上传多个文件不会保留旧筛选、字段配置、图表或工作表状态。

## 5. 字段与质量回归（核心）

- [ ] 数值、分类、日期和 ID 字段识别符合测试规则。
- [ ] 百分比、货币、千位分隔符、年份和歧义日期行为正确。
- [ ] 手动字段类型修正会重算分析，并显示转换失败数量。
- [ ] `ignore` 字段不会进入统计、图表、结论或模板候选项。
- [ ] 缺失值、完全重复行和 IQR 异常值结果正确。
- [ ] 字段类型修正和筛选不会修改 `state.rows`。

## 6. 筛选与分析回归（核心）

- [ ] 分类多选、搜索、全选和清空正常。
- [ ] 数值最小值/最大值筛选包含边界值。
- [ ] 日期开始/结束筛选包含边界日期。
- [ ] 分类、数值和日期条件组合时使用 AND。
- [ ] 清除筛选恢复全部原始记录。
- [ ] 筛选后预览、统计、图表、结论、自定义分析和模板结果同步刷新。
- [ ] 数据质量报告仍保持原始数据视角。
- [ ] 没有数值字段、分类字段或日期字段时显示空状态而不报错。
- [ ] 通用和五个场景模板只使用用户映射字段，不编造业务背景。

## 7. 图表与界面

- [ ] 每次重绘前旧 Chart.js 实例已销毁，没有图表叠加。
- [ ] 数值分布、分类 Top 10、日期趋势和自定义图表显示正确。
- [ ] 空筛选结果和无有效值时显示空状态。
- [ ] 桌面端图表布局、表格滚动和主要工作流可用。
- [ ] 手机窄屏无明显横向页面溢出，表格可在容器内横向滚动。
- [ ] 浏览器控制台没有未处理异常。

## 8. 导出回归（核心）

- [ ] 筛选后 CSV 行数与当前分析行数一致。
- [ ] 去重 CSV 基于原始数据，并保留每组重复记录的第一行。
- [ ] 两种 CSV 都保留原始字段顺序并包含 UTF-8 BOM。
- [ ] 中文、Emoji、逗号、引号和换行在导出 CSV 中正确转义。
- [ ] `=`, `@` 和非纯数值 `+` / `-` 前缀不会形成电子表格公式注入。
- [ ] HTML 和 Markdown 报告可导出并包含当前统计和结论。
- [ ] HTML 报告可离线打开，图表以内嵌 PNG 显示，无外部脚本依赖。
- [ ] 导出后可以继续筛选、修改字段和再次分析。
- [ ] 文件名不含非法字符或双向文本控制字符。

## 9. 安全与隐私（核心）

- [ ] 仓库中不存在密钥、Token、账号密码或私有 URL。
- [ ] 上传数据不会通过 `fetch`、XHR 或第三方 API 发送到服务器。
- [ ] 写入 `innerHTML` 的用户字段和值经过 `escapeHtml()`。
- [ ] Markdown 和 HTML 报告对用户内容进行对应转义。
- [ ] 新增 CDN 或外部请求已经记录用途和隐私影响。

## 10. GitHub Pages 与发布后检查

- [ ] GitHub Pages 来源分支和目录配置正确。
- [ ] CI workflow 没有部署步骤，不会覆盖 Pages 发布设置。
- [ ] 在线首页、`style.css`、`script.js`、示例 CSV 和截图资源返回 200。
- [ ] 在线演示可以加载 `sample-data.csv`。
- [ ] 在线演示可以上传一份中文 CSV 和一份多工作表 Excel。
- [ ] README Live Demo 指向当前仓库的 Pages 地址。
- [ ] 发布后检查 GitHub Actions 和 Pages 部署状态均成功。
